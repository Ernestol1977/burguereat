Idea del sistema

Aplicación web para gestión de pedidos de una hamburguesería, donde los clientes pueden realizar pedidos, el personal de cocina los gestiona y los administradores administran productos y usuarios.

1. Stack (piden...)
Frontend
- React
- React Router → rutas dinámicas 
- Context API → estado global 

* Por qué:
SPA (Single Page Application)
Manejo de login global

Backend
- Node.js
- Express

* Por qué:
API REST simple

Base de datos
- SQLite.

* Por qué:
Para tablas por entidad 

Auth
- JWT (JSON Web Token)

* Por qué:
Manejo de sesiones + roles

2. Entidades (TABLAS)

Cumplir mínimo 3 → vamos a hacer 4

#Usuario
	id
	nombre
	email
	password
	rol (admin | cocina | cliente)

#Producto
	id
	nombre
	descripcion
	precio
	stock

#Pedido
	id
	usuario_id
	estado (pendiente | en_preparacion | listo)
	fecha

#PedidoDetalle
	id
	pedido_id
	producto_id
	cantidad
